# ZAM Goals — Decomposing the Target State

ZAM's purpose is to ensure that as AI becomes more capable, humans become more capable too. This directory decomposes the project's target state into small, consumable goals, following the "Miller's number" principle (max 7 components per level).

## Core Goals

1. **[Phase 1: Individual Symbiosis](phase-1/)** — Creating a deep, learning-focused connection between a person and their personal agent.
2. **[Phase 2: Connected Community](phase-2/)** — Scaling symbiosis to the community layer through agent-to-agent matching.
3. **[Human Elevation](human-elevation/)** — The long-term philosophical goal: automation that prevents cognitive decline and fosters growth.

## Architecture Guidelines for Goals

- **Measurable**: Each goal should eventually have a corresponding metric or "Definition of Done."
- **Evolutionary**: Goals are milestones, not static endpoints. They evolve as the kernel matures.
- **Human-Centric**: Every technical goal must serve the ultimate purpose of human development.

## Relationship to Beliefs

While the `beliefs/` folder explains **WHY** ZAM works the way it does, the `goals/` folder defines **WHAT** we are building and **WHEN** it is considered successful.
