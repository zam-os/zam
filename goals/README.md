# ZAM Goals — Decomposing the Target State

ZAM's purpose is to ensure that as AI becomes more capable, humans become more capable too. This directory decomposes the project's target state into small, consumable goals, following the "Miller's number" principle (max 7 components per level).

## Core Goals

1. **[Phase 1: Individual Symbiosis](phase-1/README.md)** — Creating a deep, learning-focused connection between a person and their personal agent.
2. **[Phase 2: Connected Community](phase-2/README.md)** — Scaling symbiosis to the community layer through agent-to-agent matching.
3. **[Human Elevation](human-elevation/README.md)** — The long-term philosophical goal: automation that prevents cognitive decline and fosters growth.

## Architecture Guidelines for Goals

- **Measurable**: Each goal should eventually have a corresponding metric or "Definition of Done."
- **Evolutionary**: Goals are milestones, not static endpoints. They evolve as the kernel matures.
- **Human-Centric**: Every technical goal must serve the ultimate purpose of human development.
- **Decomposed**: No single goal document should contain more than 7 major components.

## Relationship to Beliefs & Increments

- **`beliefs/`**: Why ZAM works the way it does (foundational premises).
- **`goals/`**: What we are building and how we define success (target state).
- **`increments/`**: How we get there (executable steps and history).
