# Context Awareness: Multi-Repo & Organizational Intelligence

**Target State**: A ZAM agent that understands the user's environment by processing personal, team, and organization-level information.

## Target Components

1. **Multi-Repo Configuration** — User settings that define paths to `personal_repo`, `team_repo`, and `org_repo`.
2. **Automated Wiki & OKR Summarization** — A skill for the ZAM agent to crawl and summarize company-level goals (OKRs, wiki) for relevance matching.
3. **Scrum Board Integration** — The ability to fetch sprint tasks and work items from external boards (e.g., Jira, Azure DevOps).
4. **Context-Aware Task Fetching** — Automatically identifying the user's next unblocked task for the current session.
5. **Organizational Goal Alignment** — Mapping individual learning goals to team needs and company OKRs.
6. **Cross-Repo Reasoning** — The agent can combine knowledge from different repositories to provide holistic guidance.
7. **Team Competence Matching** — Suggesting collaborators or experts from the team based on ZAM's internal competence metrics.
