# Dependency graphs

**Belief**: Knowledge has structure. Concepts build on other concepts in directed, non-circular relationships. This structure is objective — it belongs to the knowledge, not to any individual learner.

## Components

1. **Prerequisites are directed** — Token A requires Token B. This relationship is not symmetric: knowing B doesn't require knowing A.
2. **Shared across learners** — The dependency graph is the same for everyone. It represents the structure of the knowledge itself, not a personal curriculum.
3. **Acyclic** — Circular dependencies are not allowed. If A requires B and B requires A, the model is wrong and must be restructured.
4. **Drives blocking** — When a token is forgotten, its direct prerequisites surface for review. See [prerequisite blocking](../../forgetting/prerequisite-blocking/).
5. **Emergent, not prescribed** — The graph grows as tokens are registered and connected. It is not designed top-down by a curriculum author but emerges from real learning encounters.

## Related beliefs

- [Prerequisite blocking](../../forgetting/prerequisite-blocking/) is the mechanism that makes the dependency graph actionable.
- [Token-card separation](../token-card-separation/) — prerequisites connect tokens (shared concepts), not cards (personal state).
