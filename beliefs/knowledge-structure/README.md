# Knowledge structure

**Belief**: Knowledge is a shared graph of concepts. Learning is a personal journey through that graph. These two concerns must be separated.

## Components

1. **[Token-card separation](token-card-separation/)** — A concept and a person's relationship to that concept are fundamentally different things.
2. **[Dependency graphs](dependency-graphs/)** — Concepts build on other concepts in directed, non-circular relationships.
3. **Cognitive depth** — Each concept has an inherent cognitive level, modeled via [Bloom's taxonomy](https://en.wikipedia.org/wiki/Bloom%27s_taxonomy), which determines how it is taught and assessed.
4. **Deprecation over deletion** — Knowledge goes stale. Mark it as deprecated; don't erase it. History and context matter.
5. **Domains as categories** — Concepts belong to domains (e.g., "kubernetes", "cooking") that enable [interleaving](../learning-in-context/interleaving/) and filtering.

## Related beliefs

- [Forgetting](../forgetting/) depends on this structure — blocking works because the dependency graph exists.
- [Learning in context](../learning-in-context/) adds the temporal dimension: *when* and *where* concepts are encountered.
